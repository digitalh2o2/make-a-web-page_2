A very crucial piece and great habit programmers should develop early in their careers is making sure to properly indent their code. When adding new elements to your code you should indent, or nest, the element further in. My notes for Stage 1.3

There are a couple of important things to take away from Stage 1.3 which mainly involved CSS. CSS stands for Cascading Style Sheets and it helps us avoid repetition which will save a lot of time for larger websites (as well as headaches). For example, the CSS for this HTML has allowed me to easily make changes for all the separate sections of this page with a few rules as opposed to going through every section manually and adding the code.

It's very important to keep things in order and pay close attention to detail because the most specific rules, or child elements, take priority over their parent element. One must know to keep their code organized and structured properly otherwise with one small typo, it can ruin your document completely. That's why incorporating CSS can save one the trouble of debugging their whole document line by line when they could have separated by sections.

Here are some ways to add style to an HTML.

  • For smaller projects you can add style with {} within the head tag
  • Separate CSS file sheet. This is the most common method
  • Write inline style to each and every code. Bad method and extremely repetitive.
